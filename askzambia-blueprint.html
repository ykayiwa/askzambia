1. Vision & Scope

AskZambia is Africa's first national AI knowledge platform — where anyone can ask questions about Zambia and receive accurate, source-cited answers. It serves Zambian citizens needing government information, tourists planning visits, investors researching the economy, diaspora staying connected, and researchers seeking data.

Comparable platforms exist in Ukraine (Diia.AI) and Singapore (VICA/Ask Jamie), but no African country has launched a unified national AI knowledge platform. AskZambia is positioned to be the continent's first.

The platform is built on Retrieval-Augmented Generation (RAG), meaning the AI doesn't make up answers. Instead, it searches a curated database of verified Zambian sources and generates responses grounded in that data, with citations the user can verify.

2. System Architecture

The platform follows a modern RAG architecture with three layers: the user-facing frontend, the AI orchestration layer, and the knowledge base.

5. MVP Roadmap

A focused 12-week sprint to launch AskZambia with core functionality. Using AI coding agents, this is achievable with 1-2 developers.

Infrastructure & Database

  • Create Hetzner CX32 server, install Docker + Coolify
  • Scaffold Next.js 15 app, install all dependencies (postgres.js, better-auth, OpenRouter SDK)
  • Spin up PostgreSQL 16 + pgvector container via Docker Compose
  • Run database migrations (001_init.sql, 002_indexes.sql, 003_auth.sql)
  • Configure OpenRouter API key — set LLM_MODEL to DeepSeek V3 for testing
  • Connect GitHub repo to Coolify — verify auto-deploy on push

Python Ingestion Pipeline

  • Set up Python 3.12 environment inside ingestion/ folder
  • Build web scraper (crawl4ai) — target zambialaws.com, pacra.org.zm, zra.org.zm
  • Build PDF parser (pymupdf) — target Bank of Zambia reports and government PDFs
  • Build chunker (512 tokens, 50 overlap) and embedder (OpenAI text-embedding-3-small)
  • Run first full ingestion — verify 500+ chunks stored in PostgreSQL pgvector

Core RAG Chat Experience

  • Build RAG pipeline: question → embed → pgvector search → rerank → DeepSeek generate
  • Implement citation engine (every answer links back to source document + URL)
  • Build full-screen chat UI (/chat route) — fixed left sidebar with conversation history, main streaming response area, source citation chips. Sidebar collapses to hamburger on mobile. Model name is never shown to users.
  • Build landing page (/ route) using askzambia-landing.html prototype as reference — search bar navigates to /chat?q=QUERY
  • Add suggested questions, topic browsing, conversation memory
  • Add rate limiting (20 req/min per IP) and basic abuse prevention

Multi-language, Analytics & Expansion

  • Add Bemba, Nyanja, and Tonga language support (OpenRouter models handle multilingual)
  • Self-host PostHog on same Hetzner VM — track popular questions and topics
  • Expand knowledge base to 2,000+ chunks (tourism, economic data, education)
  • Upgrade LLM_MODEL to Claude Sonnet via OpenRouter for legal/health query accuracy
  • Set up smart model routing — cheaper model for general, Claude for law/health

Polish, Test & Launch

  • Beta testing with 50-100 Zambian users — collect feedback on answer quality
  • Tune retrieval based on real questions (adjust similarity threshold, chunk size)
  • Add thumbs up/down feedback on every answer
  • SEO: generate static pages for the 50 most common Zambia questions
  • Security audit, Hetzner firewall rules, automated daily PostgreSQL backups
  • Public launch — press outreach to Zambian media and Ministry of Technology

7. Key Technical Decisions

Why RAG, not fine-tuning?

Fine-tuning a model on Zambian data would be expensive, hard to update, and prone to hallucination. RAG keeps the knowledge base separate from the model, so you can add new government data instantly without retraining. Every answer can be traced to a specific source document.

Why WhatsApp integration matters

Zambia has high mobile penetration but lower desktop/browser usage outside urban areas. WhatsApp is the dominant communication platform. A WhatsApp bot makes AskZambia accessible to millions who might never visit a website.

Why Hetzner over Vercel + Supabase?

Beyond the cost saving (€6/month vs $45+/month), the most important reason is data sovereignty. AskZambia is a national platform — Zambian citizens' questions and government data should not live on US cloud servers by default. Hetzner gives you European-hosted infrastructure you fully own. Coolify removes the DevOps burden by providing the same push-to-deploy workflow as Vercel, but on your own server.

Why PostgreSQL + pgvector instead of a separate vector DB?

Pinecone, Weaviate, and Qdrant are unnecessary complexity at this scale. PostgreSQL with the pgvector extension handles millions of document chunks with sub-100ms similarity search. Keeping everything in one database means fewer services to maintain, fewer potential failure points, and no extra monthly fees. If you outgrow it, migrating the vectors to a dedicated DB is straightforward — but most platforms never need to.

Why OpenRouter instead of connecting to Anthropic or OpenAI directly?

OpenRouter is a single API that routes to 290+ models. The integration is written once, and switching between DeepSeek (cheap testing), Gemini Flash (budget production), and Claude Sonnet (premium quality) requires changing a single environment variable — no code changes, no re-deploys. This is critical for a project that is intentionally starting cheap and upgrading over time. It also provides automatic fallback if one provider has downtime.

Why Python for document ingestion?

The document processing ecosystem in Python is significantly more mature than in Node.js. Libraries like pymupdf, crawl4ai, and unstructured handle government PDFs, scanned documents, and multi-format archives far more reliably than JavaScript equivalents. The Python ingestion scripts run as standalone jobs — they are completely separate from the Next.js app and share only the PostgreSQL database.

8. Alignment with Zambia's National AI Strategy

Zambia's National AI Strategy 2024-2026 (launched by the Ministry of Technology and Science) identifies six pillars. AskZambia directly supports three of them:

Pillar 2: AI Infrastructure & Digital Public Goods

AskZambia would be Zambia's first AI-powered digital public good, making national knowledge freely accessible.

Pillar 3: AI Talent & Capacity Building

Building and maintaining the platform creates hands-on AI engineering experience for Zambian developers.

Pillar 5: AI for Public Service Delivery

Citizens can get instant answers about government services instead of visiting offices or making calls.

Strategic recommendation: Approach the Ministry of Technology and Science early. Position AskZambia as a flagship implementation of their AI Strategy. Government endorsement provides legitimacy, access to official data, and potential funding through the national AI strategy budget.
